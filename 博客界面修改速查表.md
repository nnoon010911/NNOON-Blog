# 博客界面修改速查表

这份文档放在 Hexo 主目录，方便快速定位“想改什么，应该去改哪个文件”。

## 1. 先记住这几个最常用文件

### 站点信息

- `_config.yml`

控制：

- 网站标题 `title`
- 作者 `author`
- 网站简介 `description`
- 语言 `language`
- 时区 `timezone`
- 站点地址 `url`

适合修改：

- 网站名字
- 作者名
- 中文/英文切换
- 站点基础描述

### 主题配置

- `themes/butterfly/_config.yml`

控制：

- 导航菜单 `menu`
- 首页副标题 `subtitle`
- 首页布局 `index_layout`
- 侧边栏模块 `aside`
- 页脚 `footer`
- 配色 `theme_color`
- 字体 `font`
- 固定导航、深色模式等主题开关

适合修改：

- 导航栏菜单项
- 首页展示方式
- 侧边栏显示哪些卡片
- 主题主色、链接色、字体

### 首页结构

- `themes/butterfly/layout/includes/header/index.pug`

控制：

- 首页第一屏标题
- 首页副标题区域
- 首页首屏按钮

适合修改：

- 首页头部文案
- “开始阅读 / 关于我” 这类按钮
- 首页首屏内容结构

### 导航结构

- `themes/butterfly/layout/includes/header/nav.pug`

控制：

- 导航栏左侧站点名
- 搜索按钮
- 菜单结构输出位置

适合修改：

- 导航栏布局
- 导航栏左中右结构

### 首页文章列表样式

- `themes/butterfly/source/css/_page/homepage.styl`

控制：

- 首页文章卡片大小
- 标题字号
- 摘要长度与间距
- 卡片 hover 效果

适合修改：

- 首页文章卡片更大或更小
- 首页更像博客流还是作品集

### 首页/导航首屏样式

- `themes/butterfly/source/css/_layout/head.styl`

控制：

- 顶部导航栏
- 首页 banner / 首屏
- 首页按钮
- 文章页顶部区域

适合修改：

- 导航栏高度、背景、透明度
- 首页第一屏高度
- 首屏按钮样式

## 2. 按目标找文件

### 改网站名字、作者、简介

改：

- `_config.yml`

### 改菜单导航

先改：

- `themes/butterfly/_config.yml`

如果菜单结构要大改，再改：

- `themes/butterfly/layout/includes/header/nav.pug`
- `themes/butterfly/layout/includes/header/menu_item.pug`

### 改首页第一屏

改结构：

- `themes/butterfly/layout/includes/header/index.pug`

改样式：

- `themes/butterfly/source/css/_layout/head.styl`

### 改首页文章卡片

改结构：

- `themes/butterfly/layout/includes/mixins/indexPostUI.pug`

改样式：

- `themes/butterfly/source/css/_page/homepage.styl`

### 改侧边栏显示哪些块

先改：

- `themes/butterfly/_config.yml`

看 `aside` 下的：

- `card_author`
- `card_recent_post`
- `card_categories`
- `card_tags`
- `card_archives`
- `card_webinfo`

如果要改顺序或彻底删除输出，再改：

- `themes/butterfly/layout/includes/widget/index.pug`

### 改作者卡片

改结构：

- `themes/butterfly/layout/includes/widget/card_author.pug`

改样式：

- `themes/butterfly/source/css/_layout/aside.styl`

### 改文章页阅读体验

改：

- `themes/butterfly/source/css/_layout/post.styl`

适合修改：

- 正文宽度
- 段落间距
- 标签样式
- 引用块
- 版权块

### 改页脚

改结构：

- `themes/butterfly/layout/includes/footer.pug`

改样式：

- `themes/butterfly/source/css/_layout/footer.styl`

### 改全站颜色和字体

先改配置：

- `themes/butterfly/_config.yml`

再改变量：

- `themes/butterfly/source/css/var.styl`

如果是全局阴影、背景、链接等通用风格，再看：

- `themes/butterfly/source/css/_global/index.styl`
- `themes/butterfly/source/css/_global/function.styl`

## 3. 现在这套站点，你最可能会改的文件

如果你只是继续沿着当前风格微调，优先看这几个：

- `_config.yml`
- `themes/butterfly/_config.yml`
- `themes/butterfly/layout/includes/header/index.pug`
- `themes/butterfly/layout/includes/widget/index.pug`
- `themes/butterfly/layout/includes/widget/card_author.pug`
- `themes/butterfly/source/css/_layout/head.styl`
- `themes/butterfly/source/css/_page/homepage.styl`
- `themes/butterfly/source/css/_layout/aside.styl`
- `themes/butterfly/source/css/_layout/post.styl`
- `themes/butterfly/source/css/_layout/footer.styl`

## 4. 内容页面在哪改

### 文章

- `source/_posts/`

### 关于页

- `source/about/index.md`

### 归档页

- `source/archives/index.md`

### 分类页

- `source/categories/index.md`

### 标签页

- `source/tags/index.md`

## 5. 修改后怎么预览

在项目根目录执行：

```powershell
npm run server
```

然后打开：

```text
http://localhost:4000
```

如果只想重新生成静态页面：

```powershell
npm run build
```

不建议直接双击 `public/index.html` 预览，因为 `file:///` 路径下资源和链接表现会不准确。

## 6. 一个实用原则

优先顺序建议这样走：

1. 先改 `_config.yml`
2. 再改 `themes/butterfly/_config.yml`
3. 样式问题去改 `.styl`
4. 只有结构不够用时才改 `.pug`

这样最省事，也最不容易改乱。
